# Recipe App - Planning Document

## Overview
A minimal web app for generating recipes based on available ingredients, using Venice.ai for AI generation. Built for personal use (wife's recipe assistant).

## Tech Stack
- **Frontend**: SvelteKit (Svelte + routing)
- **AI**: Venice.ai API (text + image generation)
- **Storage**: Browser localStorage
- **Deployment**: Local development only (for now)

## Core User Flow
```
Enter Ingredients в†’ Generate Recipe Names в†’ Select One в†’ View Full Recipe
```

### Detailed Flow
1. User enters ingredients they have on hand
2. AI generates 3-5 recipe name suggestions
3. User selects one that sounds good
4. AI generates full recipe with instructions
5. User can save recipe or add ingredients to shopping list

## Pages/Views

### 1. Home Page (`/`)
- Simple ingredient input form
- Text area (one ingredient per line or comma-separated)
- "Find Recipes" button
- Optional: dietary preference toggles (vegetarian, gluten-free, etc.)

### 2. Recipe Selection (`/recipes`)
- Display 3-5 recipe name options generated by AI
- Brief description for each
- Click to select and proceed to detail

### 3. Recipe Detail (`/recipes/[id]`)
- Recipe name and generated image
- Ingredients list (with quantities)
- Step-by-step instructions
- "Add to Shopping List" button
- "Save Recipe" button
- "Start Over" button

### 4. Saved Recipes (`/saved`)
- List of saved recipes
- Click to view details
- Delete option

### 5. Shopping List (`/shopping`)
- Aggregated ingredients from selected recipes
- Check off items as purchased
- Clear list option

## Data Models

```typescript
interface Recipe {
  id: string
  name: string
  description: string
  ingredients: string[]
  instructions: string[]
  imageUrl: string
  createdAt: string
}

interface ShoppingItem {
  ingredient: string
  checked: boolean
  recipeIds: string[]
}
```

## API Integration (Venice.ai)

### Endpoints to Implement

1. **Generate Recipe Names**
   - Input: ingredients list, dietary preferences
   - Output: 3-5 recipe names with brief descriptions
   - Endpoint: Venice.ai chat completion

2. **Generate Full Recipe**
   - Input: selected recipe name, ingredients
   - Output: structured recipe (ingredients, instructions)
   - Endpoint: Venice.ai chat completion

3. **Generate Recipe Image**
   - Input: recipe name
   - Output: image URL
   - Endpoint: Venice.ai image generation

### API Key Handling
- Store in `.env` file as `VENICE_API_KEY`
- Access via SvelteKit server-side endpoints
- Never exposed to client

## LocalStorage Schema

| Key | Type | Description |
|-----|------|-------------|
| `savedRecipes` | Recipe[] | Saved recipe collection |
| `shoppingList` | ShoppingItem[] | Current shopping list |
| `dietaryPreferences` | string[] | User's dietary restrictions |
